# 测试分析（登录模块）

## 1. 项目简介
- 项目名称：登录模块测试项目
- 测试对象：手机号 + 短信验证码 + 滑块验证 + 协议勾选 + 兼容/界面检查
- 输出物：
  - 用例表：`testcases/login-test-project.xlsx`
  - 覆盖图（思维导图）：`docs/images/login-test-project.png`
  - 缺陷示例：`docs/bug-samples.md`

## 2. 测试范围
### 2.1 功能范围（包含）
- 手机号校验：已注册/未注册/位数不对/空/包含字母
- 短信验证码：正确、为空、错误、过期、倒计时、重复发送只认最新
- 滑块验证：成功、失败多次、超过次数限制、弱网/断网无法弹出
- 协议：未勾选拦截、协议链接可打开可阅读
- 兼容：谷歌/Edge/火狐/苹果
- 界面排版：文案/图片/图标与原型一致

### 2.2 非范围（不包含）
- 后端接口安全/加密/风控策略（如黑名单、设备指纹）
- 性能压测（大量并发发送验证码）
- 自动化测试脚本

## 3. 测试环境
- 操作系统：Windows 10
- 浏览器：谷歌/Edge/火狐/苹果
- 网络：正常 / 弱网 / 断网（飞行模式）
- 测试账号/手机号：
  - 已注册手机号：13200000000（示例）
  - 未注册手机号：13212345678（示例）
- 备注：短信验证码以实际收到为准

## 4. 测试策略与方法
- 输入类：等价类、边界值、错误推测
- 过程类：前置条件缺失（不发送就登录）、重复操作（重复发送/重复登录）、顺序颠倒（先输码后发送/换号用旧码）
- 时间类：倒计时、过期、重试限制
- 状态验证：按钮置灰/恢复、只认最新一次验证码、提示文案与高亮定位

## 5. 覆盖点清单
- 手机号：login_001 ~ login_006
- 短信验证码：login_007 ~ login_009、login_023 ~ login_025
- 协议：login_010、login_026
- 滑块：login_011 ~ login_017
- 兼容/界面：login_018 ~ login_022

## 6. 风险点与关注点
- 验证码风控：频繁发送限制、只认最新验证码是否生效
- 计时器与刷新：刷新后倒计时是否异常/可被绕过
- 弱网体验：滑块弹窗加载失败、按钮状态混乱
- 文案与交互：错误提示不清晰、输入框未定位导致用户迷惑

## 7. 测试结论
- 用例数量：共 26 条
- 主要覆盖：输入校验 + 关键流程 + 时间/状态/重复操作 + 兼容/界面
- 当前结论：未接入真实系统，仅做用例与分析演示
